## Historical Plans

Most files in this directory are dated execution plans for work that has already landed. Keep them for traceability, but do not treat them as the active backlog unless they are referenced by the current task board.

Examples:

- `CROSS_PLATFORM_VULKAN_*`: Vulkan implementation/gate history.
- `CROSS_PLATFORM_PHASE*`: cross-platform phase execution history.
- `PHASE1_CROSS_PLATFORM_TODO.md`: completed strategy extraction checklist.
- `OPENGL_NEXT_STAGE_TOP10.md`: completed OpenGL task batch plus historical follow-up notes.
- `MPC_HC_ITERATION_PLAN.md`: early planning baseline, superseded by current source and current gap analysis.

## Update Rule

- Keep the active backlog in `CROSS_PLATFORM_MASTER_TASKLIST.md`.
- Add separate planner files only for substantial implementation work.
- Move detailed validation results to `docs/reports/`.
