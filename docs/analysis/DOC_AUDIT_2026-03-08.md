## 3. 巡检方法

主要通过以下方式核对文档与实际仓库状态：

- 扫描高风险关键词：
  - `video_decoder`、`audio_decoder`
  - `VideoDecodeThread`、`AudioDecodeThread`、`SyncManager`
  - `playLoop`、`packet_reader`、`decoder_worker`
  - `USE_NEW_PLAYER_CORE`
  - `SDL2_DIR`、`FFMPEG_DIR`
  - `当前阶段`、`下一步计划`、`骨架/未接入`
- 对照当前仓库真实结构：
  - `include/core/*`
  - `src/core/*`
  - `include/decoder/*`、`src/decoder/*`
  - `include/render/*`、`src/render/*`
  - `CMakeLists.txt`
- 区分三类情况：
  - 需要纠正的“当前口径错误”
  - 应保留但需标明阶段的“历史内容”
  - 本就应保留的“历史记录 / 规格快照”

## 4. 当前主链基线

截至 `2026-03-08`，仓库当前播放器主链统一按以下结构理解：

- `VideoPlayer`：对外 Facade
- `PlayerCore`：核心控制面
- `Scheduler`：解码 / 渲染调度
- `core/*`：时钟、帧、队列、线程等核心能力
- `decoder/*`：解码后端与工厂
- `render/*`：SDL / D3D11 / OpenGL 渲染后端
- `playlist/*`、`subtitle/*`、`input/*`、`filters/*`：外围能力模块

因此，早期 `video_decoder` / `audio_decoder` / `playLoop` / `VideoDecodeThread` 等命名不再代表当前仓库结构，只能在“历史基线”或“历史问题分析”语境下出现。
